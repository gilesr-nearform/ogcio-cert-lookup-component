# Design & UX decisions

Short, dated rationales for the non-obvious product decisions in this
prototype. Use these as a quick brief before stakeholder meetings or
when onboarding someone new to the work.

| # | Decision | Status |
|---|----------|--------|
| [0001](./0001-ppsn-input-pattern.md) | PPSN input pattern: one empty field, no defaults, optional "Use my own PPSN" link | Accepted |

## Why this folder exists

The prototype has gone through several iterations of the PPSN input
pattern (dropdown → link-with-default → empty-field-with-helper). Each
shift had a reason worth writing down once. This folder is the
load-bearing version of those reasons — short enough to skim, sourced
enough to defend.

If you change a decision, add a new file rather than overwriting an
old one, and note "Superseded by 0007" or similar in the old file's
Status line. The history is the value.
